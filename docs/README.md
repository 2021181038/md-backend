# 문서 (Documentation)

이 폴더에는 프로젝트의 개선 사항, 아키텍처, 그리고 개발 가이드라인에 대한 문서가 포함되어 있습니다.

## 📚 문서 목록 (Document List)

### 1. [IMPROVEMENTS.md](./IMPROVEMENTS.md)
코드베이스 전반에 걸친 개선 사항을 우선순위별로 정리한 문서입니다.
- 높은 우선순위: 중복 코드 제거, 에러 처리 개선, 환경 변수 검증
- 중간 우선순위: 타입 안정성, 하드코딩된 값 제거, 성능 최적화
- 낮은 우선순위: 테스트 코드, 문서화, 코드 품질

### 2. [ARCHITECTURE.md](./ARCHITECTURE.md)

- 아키텍처 패턴
- 데이터 흐름
- 개선 제안

### 3. [CONSTANTS.md](./CONSTANTS.md)

- 현장구매 업로드 가격 계산 공식

## 🚀 빠른 시작 (Quick Start)

### 즉시 개선 가능한 항목
1. **중복 코드 제거**: 날짜 유틸리티 함수 통합 (30분)
2. **상수 파일 생성**: 하드코딩된 값 통합 (1시간)
3. **환경 변수 검증**: 앱 시작 시 검증 로직 추가 (1시간)

### 단기 개선 항목 (1-2주)
1. **Toast 시스템 도입**: alert() 대체 (2-3시간)
2. **에러 로깅 시스템**: console.error 개선 (2시간)
3. **성능 최적화**: React.memo 적용 (3-4시간)

## 📝 참고 사항

- 각 개선 사항은 독립적으로 진행 가능합니다
- 우선순위는 프로젝트 요구사항에 따라 조정 가능합니다
- 큰 변경사항은 별도 브랜치에서 작업 후 PR로 검토하는 것을 권장합니다
