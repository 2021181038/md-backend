## 마진계산기

이 문서는 **마진계산기 탭이 전체적으로 어떻게 돌아가는지**를 최대한 간단하게 설명합니다.

---

## 1. 한 줄 요약

> **주문 CSV + 정산 CSV를 업로드해서, 옵션별 수량을 합치고 → 정산/결제 금액 범위를 계산한 다음 → 원가·관세·대리비를 반영해 옵션별·전체 마진을 계산하는 도구**입니다.

---

## 2. 화면 구조 기준으로 이해하기

- **파일 업로드 영역** (`FileUploadSection`)
  - 주문 CSV 업로드
  - 정산 CSV 업로드
- **상품 선택 영역** (`ProductSelectionSection`)
  - 주문 CSV에서 읽은 상품명 목록
  - 마진을 계산할 상품만 체크해서 선택
- **액션 버튼 영역** (`ActionButtonsSection`)
  - 요약 만들기, 정산 매칭, 전체 마진 계산, 텍스트 추출 버튼
- **텍스트 영역** (`TextExtractSection`)
  - 옵션 + 수량을 텍스트로 정리해서 보여줌
- **결과 영역**
  - 정산 매칭 전: `SummarySection` (옵션별 수량만)
  - 정산 매칭 후: `ResultTable` (옵션별 마진 상세)
  - `TotalMarginSection` 에서 전체 마진 합계 표시

화면에 보이는 블록 순서대로 따라가면 **흐름 자체가 눈에 보이도록** 설계되어 있습니다.

---

## 3. 단계별 동작 요약

### 3-1. 주문 CSV 업로드

- 훅: `useMarginCalculator.handleFileUpload`
- 유틸: `parseOrderCSV`, `extractProductNames`
- 동작:
  1. 주문 CSV 파일을 읽어서 `csvData`에 저장
  2. `상품명` 컬럼을 모아서 **중복 제거 + 가나다 정렬**
  3. `productNames` 상태에 넣어 **상품 선택 리스트**로 사용

### 3-2. 정산 CSV 업로드

- 훅: `useMarginCalculator.handleSettlementUpload`
- 유틸: `parseSettlementCSV`
- 동작:
  - 정산 CSV 파일을 읽어서 `settlementData`에 그대로 보관  
    (나중에 옵션별 최저/최고 정산금, 결제금을 계산할 때 사용)

### 3-3. 계산할 상품 선택

- 훅: `toggleSelect`
- 컴포넌트: `ProductSelectionSection`
- 동작:
  - 상품명 리스트에서 체크된 것만 `selectedNames`에 저장
  - 이후 단계(요약/정산 매칭)는 **선택된 상품들만 대상으로** 진행

### 3-4. 옵션별 수량 요약 만들기

- 버튼: “요약 만들기” (`handleSummarize`)
- 유틸: `summarizeOrders`
- 동작:
  1. 주문 데이터(`csvData`) 중에서 `selectedNames`에 포함된 상품만 필터링
  2. 같은 옵션(`옵션정보`)끼리 수량을 모두 더해서 한 줄로 합침
  3. `[01]`, `[02]` 같은 번호 기준으로 정렬
  4. 결과를 `summary` 상태에 저장

이 시점에서 **“이 옵션이 총 몇 세트 팔렸는지”**까지 알 수 있습니다.

### 3-5. 정산 데이터 매칭

- 버튼: “정산 매칭” (`handleMatchSettlement`)
- 유틸: `matchSettlement`
- 동작:
  1. `summary`의 각 옵션에 대해, 정산 CSV(`settlementData`)에서 **같은 옵션정보**를 가진 행들을 찾음
  2. 가능하면 **수량이 1인 행들만** 사용해서 1세트 기준 정산금/결제금을 비교
  3. 여러 건이 있으면
     - `minSettle` / `maxSettle`: 정산금액 최소/최대
     - `minPay` / `maxPay`: 상품결제금 최소/최대
  4. 이 정보를 `summary`에 합쳐서 `matchedSummary`로 저장

이후 화면에서는:
- `matchedSummary`가 있으면 → `ResultTable`을 보여주고
- 없으면 → 단순 `SummarySection`만 보여줍니다.

### 3-6. 옵션 텍스트 추출 (부가 기능)

- 버튼: “텍스트 추출” (`handleExtractText`)
- 유틸: `extractText`
- 동작:
  - `summary`를 기반으로, `OPTION:`, `TYPE:` 같은 노이즈를 지우고  
    `옵션명 수량` 형태의 **깔끔한 텍스트 목록**을 만들어서 `extractedText`로 저장
  - 카톡/문서에 붙여넣기 좋은 형태로 보기만 하는 용도

## 4. 마진 계산에 필요한 설정들

### 4-1. 원가 입력 (`costs`)

- 위치: `ResultTable`의 **원가(₩)** 열
- 동작:
  - 옵션별로 원가(원화)를 직접 입력하면 `costs[옵션]`에 저장
  - 마진 계산 시 이 값을 기반으로 **엔화 비용**을 계산

### 4-2. 분할 값 (`divideMap`)

- 위치: `ResultTable`의 **분할(N)** 열
- 쓰임새:
  - 어떤 옵션이 “3세트 합계 금액”으로 팔렸다면 `N=3`으로 입력
  - 정산/결제 금액을 N으로 나눠서 **1세트 기준 금액**으로 계산

### 4-3. 대리비 적용 (`proxyApplied`)

- 위치: `ResultTable`의 **대찍 대리비(₩)** 열(체크박스)
- 로직:
  - 체크되면 해당 옵션에 대해 **대리비 단계**를 자동 계산
    - 원가(원) 기준으로 구간을 나누어 1,000/2,000/3,000원 등 적용
  - 마진 계산 시 이 원화 대리비를 엔화로 환산해서 빼줌

### 4-4. 관세 설정 (`dutyApplied`)

- 위치: `ResultTable`의 **옷 관세** 열(체크박스)
- 로직:
  - 평균 결제금이 `AUTO_DUTY_THRESHOLD`를 넘으면 **자동 관세(5%)**
  - 옷 관세 체크 시 **추가 12%** 적용
  - 두 조건이 함께 걸리면 합산 비율로 관세 금액을 계산

---

## 5. 옵션별 마진 계산 로직

- 핵심 함수: `calculateMargin` (`marginUtils.js`)
- 입력:
  - `row`: 한 옵션에 대한 정보 (수량, min/max 정산/결제금 등)
  - `costs`: 옵션별 원가(원화)
  - `exchangeRate`: 환율 (ONLINE)
  - `divideMap`: 분할 값
  - `proxyApplied`: 대리비 적용 여부

### 계산 순서 (한 옵션 기준)

1. **원가(₩ → ¥)** 변환
2. `min/max` 정산금·결제금의 **평균값** 계산  
   - 분할 값이 있으면 `÷ N` 해서 1세트 기준으로 환산
3. 자동 관세 + 옷 관세 설정에 따라 **관세 금액** 계산
4. 대리비 체크 여부에 따라 **대리비(원화) → 엔화**로 환산
5. `마진(평균) = 평균 정산금 - 원가(¥) - 관세 - 대리비(¥)`
6. `총 마진 = 마진(평균) × 수량`

### 결과 값

- `costWon`: 원가(원)
- `costYen`: 원가(엔)
- `avgSettle`: 평균 정산금(엔)
- `avgPay`: 평균 결제금(엔)
- `marginAvg`: 1세트당 마진(엔)
- `totalMarginAvg`: 총 마진(엔)
- `proxyFee`: 대리비(원)

이 값들을 `ResultTable`이 한 줄씩 보여줍니다.

---

## 6. 전체 마진 합계

- 핵심 함수: `calculateTotalMargin` (`marginUtils.js`)
- 입력:
  - `matchedSummary`, `costs`, `exchangeRate`, `dutyApplied`, `proxyApplied`
- 동작:
  1. 각 옵션에 대해 **평균 마진 × 수량**을 모두 합산
  2. 대리비도 옵션별·수량별로 모두 합쳐 **총 대리비(원)** 계산
  3. 전체 마진(엔)과 환율을 곱한 **대략적인 원화 마진**을 구함
- 출력:
  - `total`: 최종 합산 마진(엔)
  - `totalWon`: 환율을 곱한 원화 기준 마진(대략)
  - `totalProxyFee`: 총 대리비(원)

이 값은 화면의 `TotalMarginSection`에서 한 번에 확인할 수 있습니다.

---

## 7. CSV 내보내기

- 버튼: “CSV로 내보내기” (`handleExportCSV`)
- 유틸: `exportToCSV`
- 동작:
  - `matchedSummary` + `calculateMargin` 결과를 기반으로
  - 옵션별 마진 정보가 담긴 CSV 파일을 생성
  - 이후 엑셀 등에서 추가 분석하거나 기록용으로 사용 가능

---

## 8. 최종 요약

- **입력**: 주문 CSV, 정산 CSV, 옵션별 원가/분할값/대리비/관세 설정
- **처리**:
  - 옵션별로 수량 합치기
  - 정산/결제 금액의 최소·최대 → 평균값 계산
  - 원가·관세·대리비를 반영해 옵션별 마진 계산
- **출력**:
  - 옵션별 마진 표 (`ResultTable`)
  - 전체 마진/총 대리비 요약 (`TotalMarginSection`)
  - CSV 파일로 결과 저장

이 정도만 이해하고 있으면, 세부 구현을 몰라도 **마진계산기가 어떤 데이터 흐름으로 돌아가는지**를 충분히 파악할 수 있습니다.
